3. SUNRISE & SUNSET TIMES
==================================================================================

Current: astronomy-engine ‚úì

FREE ALTERNATIVES (as backup/verification):
-------------------------------------------

A) Sunrise-Sunset.org API (RECOMMENDED - Simplest)
   - URL: https://api.sunrise-sunset.org/json?lat={LAT}&lng={LNG}&date={YYYY-MM-DD}
   - API Key: ‚ùå NO KEY NEEDED
   - Limits: Unlimited, completely free
   - Response Format: JSON
   - Example:
     https://api.sunrise-sunset.org/json?lat=13.0827&lng=80.2707&date=2025-12-23
   - Response:
     {
       "results": {
         "sunrise": "6:24:15 AM",
         "sunset": "5:54:32 PM",
         "solar_noon": "12:09:23 PM",
         "day_length": "11:30:17"
       },
       "status": "OK"
     }

B) Open-Meteo Weather API
   - URL: https://api.open-meteo.com/v1/forecast?latitude={LAT}&longitude={LNG}&daily=sunrise,sunset
   - API Key: ‚ùå NO KEY NEEDED
   - Limits: Free, unlimited for non-commercial
   - Response Format: JSON
   - Example:
     https://api.open-meteo.com/v1/forecast?latitude=13.08&longitude=80.27&daily=sunrise,sunset&timezone=Asia/Kolkata
   - Documentation: https://open-meteo.com/en/docs

C) USNO API (US Naval Observatory)
   - URL: https://aa.usno.navy.mil/api/rstt/oneday?date={MM/DD/YYYY}&coords={LAT},{LNG}
   - API Key: ‚ùå NO KEY NEEDED
   - Limits: Free but rate-limited (sometimes slow)
   - Most Accurate: Yes (official US Navy data)
   - Example:
     https://aa.usno.navy.mil/api/rstt/oneday?date=12/23/2025&coords=13.0827,80.2707

D) IPGeolocation API (HAS FREE TIER)
   - URL: https://api.ipgeolocation.io/astronomy?apiKey={KEY}&lat={LAT}&long={LNG}
   - API Key: ‚úÖ YES - FREE KEY NEEDED
   - Limits: 1,000 requests/day (Free tier)
   - Sign up: https://ipgeolocation.io/signup.html
   - Features: Sunrise, sunset, moonrise, moonset, moon phase, solar noon
   - Response includes: current_time, sunrise, sunset, day_length, moon_phase

A: ‚ö†Ô∏è YES! Regional differences exist:

   - North India: Uses different month names (Chaitra, Vaishakha, etc.)

   - Keep same astronomical calculations (universal)
   - Change naming conventions (region-specific)

Q: What about local festivals/holidays?
A: Not covered by these APIs. You'd need a separate database of:
